// problem
The problem

Tipping creators is broken

Web2 tipping

Twitter Tips, Patreon, Ko-Fi.

  • Both sides must sign up
  • 2–30% in fees
  • Payouts take days

Web3 tipping

Chain-native transfers.

  • Recipient needs a wallet on the right chain
  • Gas often costs more than the tip
  • No in-feed gesture

Result: tipping is a friction-filled chore almost no one does.

02
// solution
The solution

Tip any X handle in one click

  • Tip anyone — even non-crypto users. Tips to unregistered handles are held in an on-chain escrow, keyed by the handle.
  • One-click claim. Recipient signs up once (Privy / wallet), the backend binds their wallet on-chain, funds land instantly.
  • Tip from inside X. A Chrome extension injects a Tip button into the feed and onto profiles.
  • Two modes, one contract. Direct tip (extension/web) and Twitter-command tip via an AI bot.

// mode A
How it works — Mode A

Direct tip (extension / web)

Click Tip on X→ Escrow.tip(handle)→ registered?

Registered recipient

STT transfers directly to their wallet, in < 1 second.

Unregistered recipient

STT locked in escrow under the handle. Backend indexes a claim; recipient claims after sign-up.

The user's own wallet signs the transaction (Privy embedded wallet on web, MetaMask in the extension).

05
// mode B
How it works — Mode B + claim

Tip by tweet, claim with one sign-up

Mode B — Twitter command

@xeniaBot tip @bob 5 STT
  • Pre-deposit + authorize the bot
  • Bot parses intent with Gemini
  • Calls tipOnBehalf() — bot can only forward, never withdraw

Claim flow

  • Recipient signs up (Privy)
  • Backend binds handle → wallet on-chain (immutable)
  • claim(handle) pulls all pending escrow tips

// security
Security

Built so funds can't be touched by the wrong party

  • nonReentrant on every value-moving call
  • Fees tracked separately from user funds
  • Wallet binding immutable once set
  • Bot delegation scoped to forwarding only
  • 90-day sender refund safety net
  • Two-step ownership transfer

The backend never holds user keys — Privy embedded wallets stay client-side.
